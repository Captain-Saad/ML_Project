# Intelligent Real Estate Analytics & Price Intelligence

## 1. Project title

**Housing Intelligence Platform** вЂ” geospatial + NLP augmented valuation, tiering, explainable ranking, and interactive Streamlit UI for residential listings.

## 2. Project overview

This repository implements an end-to-end analytics and machine learning workflow on a cleaned US-style listings table (~6.5k rows). The system predicts list prices, enriches rows with text-derived investment signals, clusters listings geographically, assigns percentile-based quality tiers (3-tier system), scores every home on a transparent 0вЂ“100 **HouseScore**, surfaces undervalued opportunities, and documents model behavior with SHAP. 

A premium **Streamlit web application** is provided for interactive exploration.

The entire workflow is reproducible from a single entrypoint: `python src/pipeline.py`.

## 3. Features

- **Robust preprocessing** with median/mode imputation, safe text handling, engineered ratios, geo clustering, and lot-size normalization using `keyFacts_combined` cues.
- **Target encoding** (`category_encoders`) for `city`, `state`, and `zip` plus **RobustScaler** on the modeling feature block.
- **NLP stack**: keyword-derived luxury/renovation/view/amenity/condition signals, HOA extraction, TFвЂ“IDF + TruncatedSVD dimensions, and TextBlob sentiment on `listingRemarks`.
- **Expanded Model zoo**: `RandomizedSearchCV` tuning for 8 models including Linear Regression, Ridge, Decision Tree, Random Forest, Gradient Boosting, XGBoost, SVR, and Deep Learning (MLP).
- **HouseScore & 3-Tier Classification** built from a weighted blend of predicted price, luxury, geo desirability, amenities, lot size, and new-construction status (all MinвЂ“Max normalized before weighting). Homes are classified into **Budget**, **Mid-Range**, and **Premium** based on 33rd/67th percentiles.
- **Investment analytics** using percentage pricing gaps and ranked undervalued scores with exports.
- **Visualization suite** (matplotlib + seaborn + Plotly HTML maps) and **SHAP** explainability plots.
- **Premium Streamlit Dashboard** with a modern UI/UX, glassmorphism design, interactive metrics, and integrated visualizations.

## 6. How to run

**1. Run the ML Pipeline:**
```bash
python -W ignore src/pipeline.py
```
*(The `-W ignore` flag suppresses excessive warnings during Ridge cross-validation).*
Artifacts (processed data, models, plots, reports) are written automatically. The console prints stage progress, an 8-model comparison table, the best model name (e.g., SVR), the top 10 ranked homes, and leading undervalued opportunities.

**2. Launch the Web App:**
```bash
streamlit run app.py
```

## 7. ML models used

The pipeline tunes and evaluates 8 algorithms via `RandomizedSearchCV` (`cv=5` or `cv=3`, `random_state=42`):
- `LinearRegression` & `Ridge` (Interpretable baselines)
- `DecisionTreeRegressor` (Simple non-linear baseline)
- `RandomForestRegressor` (Nonlinear bagged trees)
- `GradientBoostingRegressor` & `XGBRegressor` (Stage-wise and high-capacity boosted trees)
- `SVR` (Support Vector Regressor with RBF kernel - *Current Champion*)
- `MLPRegressor` (Deep Learning Neural Network)

The champion minimizes RMSE on the held-out 20% split mirroring the preprocessing encoders. Permutation importance is used for models that do not expose native feature importances.

## 8. NLP pipeline explanation

1. **Keyword lexicons** scan `listingRemarks` + `keyFacts_combined` for luxury, renovation, view, amenity, and condition language.
2. **HOA extraction** applies regex heuristics to recover numeric HOA fees when present.
3. **TFвЂ“IDF (`max_features=100`)** is fit on the training text split only, then applied to all rows.
4. **TruncatedSVD (`n_components=20`)** compresses the sparse TFвЂ“IDF matrix into `nlp_dim_1 вЂ¦ nlp_dim_20`.
5. **TextBlob polarity** quantifies marketing tone for `listingRemarks`.

## 9. HouseScore explanation

`composite_score` blends (after per-component MinвЂ“Max scaling):

| Component | Weight | Description |
| --- | ---: | --- |
| Predicted price | 40% | Model expectation of fair value |
| Luxury score | 20% | Keyword richness for premium positioning |
| Geo desirability (`geo_desirability`) | 15% | Local listing-density proxy within the KMeans geo cluster (no price leakage) |
| Amenities score | 10% | Count of amenity-related hits |
| Lot size (sqft-normalized) | 10% | Land scale signal |
| New construction flag | 5% | Binary quality / risk signal |

`house_score = round(composite_score * 100, 2)` and tiers are assigned by composite percentiles: **Budget** (bottom 33%), **Mid-Range** (middle 34%), and **Premium** (top 33%).

## 10. Explainability section

`src/explain.py` fits a SHAP explainer (`TreeExplainer`, `LinearExplainer`, or `KernelExplainer` for Pipelines) on a stratified background sample, exports:

- Beeswarm summary (`shap_summary_beeswarm.png`)
- Bar chart (`shap_bar_importance.png`)
- Waterfall for the single highest `house_score` listing (`shap_waterfall_top_house.png`)

The script also prints the top features by mean absolute SHAP value for quick narration in writeups.
